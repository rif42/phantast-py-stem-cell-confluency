# Data Model

## Entities

### Workspace
The active environment containing the current image(s), active pipeline, and overall application state.

### Image
A single raw microscope image loaded into the application for inspection or processing.

### Pipeline
A saved sequence of processing steps designed to standardize and analyze images.

### ProcessNode
An individual operation within a pipeline (e.g., PHANTAST algorithm, crop, grayscale, text annotation) with its own specific configuration parameters.

### BatchJob
A record of a pipeline execution across a complete folder or dataset of images.

### ConfluencyResult
The calculated data output (confluency percentage, area metrics) generated by running the final pipeline on an image.

## Relationships

- Workspace has one active Pipeline and has many Images (or a folder of Images).
- Workspace has many BatchJobs.
- Pipeline has many ProcessNodes.
- ProcessNode belongs to a Pipeline.
- BatchJob belongs to a Workspace and applies one Pipeline to many Images.
- Image has one ConfluencyResult (after processing).
- ConfluencyResult belongs to an Image.
